1.A : FORMULAIRE

Quelles sont, et à quoi servent, les 3 types d’informations qu’il faut repérer dans un formulaire HTML pour pouvoir exploiter ses données à l’aide de PHP ?
  • method: Méthode d'envoie des données au script( GET ou POST )
  • action: Url du fichier PHP qui va recevoir les informations
  • name: un attribut rempli par une valeur ( Le nom du champ sera ensuite considéré par le script PHP comme une variable contenant la valeur saisie. )
Remarque:
Si l'on a affaire à un formulaire d'upload de fichiers, la balise <form> doit comporter l'attribut enctype et la valeur multipart/form-data multipart/form-data .

1.B : ARCHITECTURE CLIENT/SERVEUR

Qu’est-ce qu’une architecture client-serveur ?

C'est un mode de communication à travers un réseau entre plusieurs programmes :
  • L'un, qualifié de client, envoie des requêtes
  • L'autre(ou les autres), qualifiés de serveurs, attendent les requêtes des clients et y répondent.
Où se situent le navigateur et le PHP dans cette architecture ?

  • Le navigateur : Se trouve cote client
  • Le moteur PHP: Se trouve cote serveur
Entre le moment où j’écris du code PHP dans un traitement de texte sur mon ordinateur et le moment où j’affiche le résultat dans le navigateur, que s’est-il passé ?

  • D'abord j’envoie mon code a un serveur web( Apache,Nginx,IIS, ou Tomcat, ...etc). Ce, en utilisant le protocole HTTP.
  • Ensuite le serveur web genere une page reponse( eventuelle interrogation de se la base d'une data base)
    • Enfin le serveur retourne la page reponse à mon navigateur en utilisant le même protocole.
